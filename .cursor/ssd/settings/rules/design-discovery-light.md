# 扩展的轻量发现流程

## 目标
快速分析现有系统和集成需求，用于功能扩展。

## 聚焦发现步骤

### 1. 扩展点分析
**识别集成方法**：
- 定位现有扩展点或接口
- 确定修改范围（文件、组件）
- 检查要遵循的现有模式
- 识别向后兼容性要求

### 2. 依赖项检查
**验证兼容性**：
- 检查新依赖项的版本兼容性
- 验证API契约未更改
- 确保管道中无破坏性更改

### 3. 快速技术验证
**仅针对新库**：
- 使用WebSearch查找官方文档
- 验证基本使用模式
- 检查已知兼容性问题
- 确认许可兼容性
- 在 `research.md`（技术对齐章节）中记录关键发现

### 4. 集成风险评估
**快速风险检查**：
- 对现有功能的影响
- 性能影响
- 安全考虑
- 测试要求

## 何时升级到完整发现
如果发现以下情况，切换到完整发现：
- 需要重大架构更改
- 复杂的外部服务集成
- 安全敏感的实施
- 性能关键组件
- 未知或文档不足的依赖项

## 输出要求
- 清晰的集成方法（在 `research.md` 中注意边界影响）
- 要修改的文件/组件列表
- 带版本的新依赖项
- 集成风险和缓解措施
- 测试重点领域
